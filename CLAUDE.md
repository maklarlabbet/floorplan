## Technology

## Rules

Always use prepared statements.

Never delete existing functionality.

Never modify config.php unless asked.

Never modify .cpanel.yml unless asked.

Never modify git-deploy.php unless asked.

Never modify .htaccess unless asked.

Explain your plan before editing files.

Do not commit automatically.

Do not push automatically.

Always preserve coding style.

Create reusable functions.

Test PHP syntax before committing.





\## Deployment Rules



\- Never push to GitHub without explicit approval.

\- Never modify deployment scripts unless requested.

\- Never delete files unless instructed.

\- If a change affects multiple files, explain why before making it.

\- After coding, run a PHP syntax check on modified files.
